## 1. 分层系统架构图

从客户端到基础设施的请求路径（与 `star-pivot-controller` + `star-pivot-ui` 部署方式一致）。

```mermaid
flowchart TB
    subgraph Client["客户端层"]
        Web["star-pivot-ui<br/>Vue 3 + Vite + Element Plus"]
    end

    subgraph Entry["统一入口"]
        Tomcat["Spring Boot 内嵌 Tomcat<br/>:8080  context-path: /api"]
    end

    subgraph AppLayer["安全与过滤器链"]
        Cors["CORS 配置<br/>CorsConfigurationSource"]
        Jwt["JwtAuthenticationFilter<br/>Token + 黑名单"]

        Cors --> Jwt --> Sec
    end

    subgraph Ctrl["控制层 star-pivot-controller"]
        Ctl["REST Controller<br/>auth / system / mall / monitor ..."]
    end

    subgraph Biz["业务模块层 star-pivot-module"]
        M_sys["star-pivot-system<br/>用户/角色/菜单/认证"]
        M_dict["star-pivot-dict<br/>字典"]
        M_gen["star-pivot-generator<br/>代码生成"]
        M_qtz["star-pivot-quartz<br/>定时任务"]
        M_mon["star-pivot-monitor<br/>监控"]
        M_mall["star-pivot-mall<br/>商城"]
    end

    subgraph Svc["模块内分层"]
        S1["Service"]
        S2["Mapper / Entity"]
        S1 --> S2
    end

    subgraph Framework["框架层 star-pivot-framework"]
        FW_bs["framework-boot-starter<br/>聚合入口"]
        FW_sec["star-pivot-security"]

        FW_sec --> FW_core

        FW_sec --> FW_file
        FW_sec --> FW_web
    end

    subgraph Dep["依赖管理"]
        BOM["star-pivot-dependencies BOM"]
    end

    subgraph Infra["基础设施"]
        MySQL["MySQL<br/>业务数据"]
        Redis["Redis<br/>缓存 / JWT 黑名单 / RefreshToken"]
        OSS["MinIO / 阿里云 OSS<br/>文件与头像"]
    end

    Web --> Tomcat
    Tomcat --> Cors
    Sec --> Ctl
    Ctl --> M_sys
    Ctl --> M_dict
    Ctl --> M_gen
    Ctl --> M_qtz
    Ctl --> M_mon
    Ctl --> M_mall
    M_sys --> S1
    M_dict --> S1
    M_mall --> S1
    M_mon --> S1
    S2 --> FW_bs
    FW_bs --> BOM
    FW_sec --> Redis
    S2 --> MySQL
    FW_file --> OSS
```

说明：

| 层级 | 要点 |
|------|------|
| 客户端 | 仅 `star-pivot-ui` 为仓库内前端；通过 `/api` 访问后端 |
| 安全链 | CORS → JWT 过滤器（含黑名单）→ Spring Security；详见 security 使用说明 |
| 控制层 | **Controller 集中在 controller 模块**，业务逻辑在 module 的 Service |
| 业务模块 | system 依赖 dict；monitor 依赖 system；generator/quartz 已解耦 system |
| 框架层 | 业务侧统一依赖 **boot-starter**；已移除历史模块 `star-pivot-common` |
| 基础设施 | MySQL 持久化；Redis 缓存与令牌；OSS/MinIO 文件 |

## 3. 系统总体架构（概览）

```mermaid
flowchart LR
    subgraph 前端
        UI[star-pivot-ui]
    end
    subgraph 后端
        API[star-pivot-controller]
        MOD[star-pivot-module]
        FW[star-pivot-framework]
        API --> MOD
        MOD --> FW
    end
    subgraph 存储
        DB[(MySQL)]
        RD[(Redis)]
        FS[(MinIO/OSS)]
    end
    UI -->|HTTP /api| API
    FW --> RD
    MOD --> DB
    FW --> FS
```

- **前后端分离**：Vue 3 SPA + Spring Boot REST API。  
- **版本统一**：`star-pivot-dependencies` BOM 管理所有模块与第三方版本。  
- **可执行入口**：仅 `star-pivot-controller` 打可运行 JAR（`StarPivotApplication`）。

## 4. 认证与令牌时序

```mermaid
sequenceDiagram
    participant UI as star-pivot-ui
    participant API as AuthController
    participant Auth as AuthService
    participant AM as AuthenticationManager
    participant TS as TokenService
    participant JWT as JwtUtil
    participant RD as Redis

    UI->>API: POST /auth/login
    API->>Auth: login(用户名/密码/验证码)
    Auth->>AM: authenticate
    AM-->>Auth: LoginUser
    Auth->>TS: createTokenPair
    TS->>JWT: generateToken access
    TS->>RD: 存储 refreshToken
    TS-->>UI: accessToken + refreshToken

  Note over UI,API: 后续请求 Header: Authorization Bearer accessToken

    UI->>API: 业务 API + Bearer
    API->>API: JwtAuthenticationFilter
    API->>RD: 黑名单校验
  alt Token 有效
        API-->>UI: 200 业务数据
  else 未登录/无权限
        API-->>UI: 401 / 403 JSON
    end

    UI->>API: POST /auth/logout
    API->>TS: logout(token)
    TS->>RD: accessToken 入黑名单
    TS->>RD: 吊销 refreshToken
```

- 登录、刷新、登出接口默认 **匿名放行**（`SecurityConfig` 内置路径）。  
- 登出后 Access Token 进入 Redis 黑名单，Refresh Token 从 `jwt:refresh:user:{userId}` 删除。

## 5. RBAC 权限模型关系图

```mermaid
erDiagram
    SysUser ||--o{ UserRole : 拥有
    SysRole ||--o{ UserRole : 被分配
    SysRole ||--o{ RoleMenu : 关联
    SysMenu ||--o{ RoleMenu : 被授权
    SysUser ||--o{ UserPost : 任职
    SysPost ||--o{ UserPost : 被分配
    SysUser }o--|| SysDept : 所属

    SysUser {
        bigint user_id
        string user_name
        string password
        bigint dept_id
    }

    SysRole {
        bigint role_id
        string role_name
        string role_key
        string data_scope
    }

    SysMenu {
        bigint menu_id
        bigint parent_id
        string path
        string component
        string perms
    }

    UserRole {
        bigint user_id
        bigint role_id
    }

    RoleMenu {
        bigint role_id
        bigint menu_id
    }

    SysDept {
        bigint dept_id
        bigint parent_id
        string dept_name
    }
```

- **用户** ↔ **角色** ↔ **菜单/权限**（`perms` 对应按钮级 `hasAuthority` / `v-auth`）。  
- **数据权限**：角色 `data_scope` + `DataScopeService`，Mapper XML 中 `dataScopeFilter` 片段。  
- 权限编码约定见 [`权限编码与数据权限规范.md`](权限编码与数据权限规范.md)。

## 6. 请求处理与权限校验流程

```mermaid
flowchart TD
    A[HTTP 请求] --> B{路径在 permitAll?}
    B -->|是| Z[进入 Controller]
    B -->|否| C{携带有效 JWT?}
    C -->|否| D[401 未认证]
    C -->|是| E{Token 在黑名单?}
    E -->|是| D
    E -->|否| F[SecurityContext 写入用户]
    F --> G{方法 @PreAuthorize?}
    G -->|通过| Z
    G -->|拒绝| H[403 无权限]
    Z --> I[Service / 数据权限 / 业务逻辑]
    I --> J[返回 Result JSON 或文件流]
```

## 7. 前端路由与菜单加载流程

```mermaid
flowchart TB
    A[用户打开系统] --> B{本地是否有 token?}
    B -->|否| C[跳转登录页]
    B -->|是| D[路由守卫]
    D --> E["GET /auth/userinfo<br/>或菜单树接口"]
    E --> F{接口成功?}
    F -->|否| G[清除 token → 登录]
    F -->|是| H[写入 Pinia：用户/角色/菜单]
    H --> I[按菜单 path/component 生成动态路由]
    I --> J[router.addRoute]
    J --> K[渲染侧栏]
    K --> L[进入目标页]
    C --> M[POST /auth/login]
    M --> D
```

- 动态路由来自后端菜单树；按钮显隐使用 `v-auth` / `hasAuth`，与菜单 `perms` 一致。

## 8. 缓存与菜单/字典数据流

```mermaid
flowchart LR
    subgraph 读路径
        A1[请求菜单树/字典] --> A2{Redis 有缓存?}
        A2 -->|是| A3[返回缓存]
        A2 -->|否| A4[查 DB]
        A4 --> A5[写入 Redis]
        A5 --> A3
    end

    subgraph 写路径
        B1[菜单/字典变更] --> B2[更新 DB]
        B2 --> B3["@CacheEvict"]
        B3 --> B4[清除缓存]
    end
```

配置参考 `application.yml` 中 `star-pivot.cache.*`（如 `menu-tree-ttl`、`dict-data-ttl`）。
