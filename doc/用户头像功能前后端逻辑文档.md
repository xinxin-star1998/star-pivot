# 用户头像功能前后端逻辑文档

## 一、后端逻辑

### 1. 头像管理控制器

**文件位置**: `star-pivot-controller/src/main/java/com/star/pivot/controller/AvatarController.java`

#### 权限控制规则
- 个人中心场景：用户只能查看和修改自己的头像
- 后台管理场景：拥有"用户管理"权限（system:user:list）的管理员，只能查看其他用户的头像，不可修改
- 超级管理员（admin角色）：可以查看和修改所有用户头像
- 不允许操作系统预置的超级管理员账号头像

#### 核心接口

##### 1.1 上传头像接口
```java
@PostMapping("/upload")
public Result<Map<String, String>> upload(
    @RequestParam("file") MultipartFile file,
    @RequestParam("userId") String userId,
    @RequestParam(value = "usePresignedUrl", defaultValue = "false") boolean usePresignedUrl)
```

**功能说明**:
- 文件大小验证：限制5MB
- 文件类型验证：只允许图片文件
- 权限校验：只能上传自己的头像（超级管理员除外）
- 支持两种模式：
  - 公共读桶：返回永久访问URL
  - 私有桶：返回永久URL（存库）和预签名URL（前端展示用）

**返回数据**:
```json
{
  "avatarUrl": "https://xxx/avatar/123.png",
  "presignedUrl": "https://xxx?signature=xxx&expires=xxx",
  "isPresigned": "true"
}
```

##### 1.2 删除头像接口
```java
@DeleteMapping("/delete")
public Result<?> delete(@RequestParam("userId") String userId)
```

**功能说明**:
- 权限校验：只能删除自己的头像（超级管理员可删除所有用户头像）
- 删除该用户下所有头像文件

### 2. OSS存储工具类

**文件位置**: `star-pivot-framework/star-pivot-framework-file/src/main/java/com/star/pivot/framework/utils/OssUtil.java`

#### 核心方法

##### 2.1 上传头像
```java
public String uploadAvatar(MultipartFile file, String userId)
```

**功能特性**:
- 用户ID格式校验（仅允许数字，防止路径穿越）
- 文件大小限制：2MB
- 支持格式：JPG、PNG、GIF、WEBP
- 存储路径：`avatar/{userId}.{suffix}`
- 先上传新文件，再删除旧头像（避免上传失败时丢失旧头像）

##### 2.2 生成永久访问URL
```java
public String getPermanentUrl(String objectName)
```

**URL格式**:
- 配置了url-prefix：使用配置的前缀
- 默认格式：`https://{bucket-name}.{endpoint}/{object-name}`

##### 2.3 生成临时访问链接
```java
public String getPresignedUrl(String objectName)
```

**功能说明**:
- 有效期：默认7天
- 适用于私有桶场景
- 避免直接访问私有文件导致403错误

### 3. MinIO存储工具类

**文件位置**: `star-pivot-framework/star-pivot-framework-file/src/main/java/com/star/pivot/framework/utils/MinioUtil.java`

#### 核心方法
- `uploadAvatar(MultipartFile file, String userId)` - 上传头像（与OssUtil行为一致）
- `uploadAvatarWithUrl(MultipartFile file, String userId)` - 返回永久访问URL（公共桶）
- `uploadAvatarWithPresignedUrl(MultipartFile file, String userId)` - 返回预签名URL（私有桶）
- `deleteAvatar(String userId)` - 删除用户头像

```vue
<art-avatar-upload
  ref="avatarUploadRef"
  v-model="formData.avatar"
  :user-id="formData.userId"
  :size="100"
```

#### 参数说明

- `size`: 头像显示尺寸100px

#### 提交逻辑
```typescript
// 添加用户：先创建用户，再上传头像
if (dialogType.value === 'add') {
  const addResult = await fetchAddUser(formData)
  const newUserId = addResult?.userId || addResult?.id
  if (newUserId) {
    formData.userId = newUserId
    await avatarUploadRef.value.uploadImageToServer()
  }
} else {
  // 编辑用户：直接更新，自动上传头像
  await fetchUpdateUser(formData)
  // 若编辑的是当前登录用户，同步头像到 store
  const cur = userStore.getUserInfo as any
  const curId = cur?.userId ?? cur?.user?.userId
  if (curId != null && String(formData.userId) === String(curId)) {
    userStore.setUserInfo({
      avatar: formData.avatar,
      avatarUpdatedAt: Date.now()
    })
  }
}
```

```typescript
const updateTopAvatarDisplayUrl = async () => {
  const avatar = userDetail.value.avatar
  if (!avatar || avatar === '') {
    topAvatarDisplayUrl.value = defaultAvatar
    return
  }
  const path = extractAvatarPathFromUrl(avatar)
  const isOssUrl = avatar.includes('aliyuncs.com') || avatar.includes('.oss-')
  if (path && isOssUrl) {
    try {
      const res = (await fetchGetAvatarPresignedUrl(path)) as any
      const presigned = res?.presignedUrl ?? res?.data?.presignedUrl
      if (presigned) {
        topAvatarDisplayUrl.value = presigned
      } else {
        topAvatarDisplayUrl.value = defaultAvatar
      }
    } catch {
      topAvatarDisplayUrl.value = defaultAvatar
    }
  } else {
    topAvatarDisplayUrl.value = avatar
  }
}
```

## 三、API接口定义

**文件位置**: `star-pivot-ui/src/api/user/user.ts`

```typescript
// 上传头像
export function fetchUploadAvatar(data: FormData) {
  return request.post({
    url: '/api/avatar/upload',
    data,
    headers: {
      'Content-Type': 'multipart/form-data'
    }
  })
}

// 删除头像
export function fetchDeleteAvatar(userId: string) {
  return request.del({
    url: '/api/avatar/delete',
    params: {
      userId
    }
  })
}

// 获取头像临时访问链接
export function fetchGetAvatarPresignedUrl(filePath: string) {
  return request.get({
    url: '/api/avatar/presigned-url',
    params: {
      filePath
    }
  })
}
```

## 五、流程总结

### 5.1 上传头像流程

#### 前端流程
1. 用户选择头像文件
2. 组件进行本地预览
3. 调用 `fetchUploadAvatar` API
4. 接收返回的 `avatarUrl` 和 `presignedUrl`
5. 更新表单数据和store

#### 后端流程
1. 验证文件大小和类型
2. 验证用户权限
3. 上传文件到OSS/MinIO（路径：`avatar/{userId}.{suffix}`）
4. 删除旧头像文件
5. 返回永久URL和预签名URL

### 5.2 显示头像流程

#### 公共桶场景
- 直接使用存储的 `avatarUrl` 显示

#### 私有桶场景
1. 判断是否为OSS URL
2. 提取文件路径
3. 调用 `fetchGetAvatarPresignedUrl` 获取临时访问链接
4. 使用预签名URL显示头像

### 5.3 删除头像流程

#### 前端流程
1. 用户点击删除按钮
2. 调用 `fetchDeleteAvatar` API
3. 清空表单中的头像URL

#### 后端流程
1. 验证用户权限
2. 删除该用户下所有头像文件（`avatar/{userId}.*`）

## 六、安全机制

### 6.1 路径穿越防护
- 用户ID仅允许数字，防止 `../` 等路径穿越攻击

### 6.2 权限控制
- 严格的权限校验，用户只能操作自己的头像

### 6.3 文件类型校验
- 只允许图片格式（JPG、PNG、GIF、WEBP）

### 6.4 文件大小限制
- 2MB（OSS）/ 5MB（Controller）

### 6.5 私有桶保护
- 使用预签名URL，避免文件直接暴露

### 6.6 先传后删机制
- 上传新头像成功后再删除旧头像，防止数据丢失
