# 通用导入导出使用说明

本文档说明如何对接 StarPivot 项目的**通用导入导出**功能，供其他业务页面（如部门、角色等）复用。

---

## 一、概述

- **后端**：`CommonImportExportController` 提供统一的导入、导出、模板下载接口；各业务通过实现 `ImportExportService` 接入。
- **前端**：`api/common/import-export.ts` 提供 `fetchImportData`、`fetchExportData`、`fetchDownloadTemplate`，页面按业务类型调用即可。
- **Excel 处理**：后端使用 EasyExcel 生成/解析；前端使用 XLSX 解析上传的 Excel 为行数据后提交。

### 相关文件

| 类型 | 路径 |
|------|------|
| 后端 Controller | `star-pivot-controller/.../CommonImportExportController.java` |
| 后端服务接口 | `star-pivot-system/.../ImportExportService.java` |
| 服务工厂 | `star-pivot-system/.../ImportExportServiceFactory.java` |
| 前端 API | `star-pivot-ui/src/api/common/import-export.ts` |
| 文件工具 | `star-pivot-ui/src/utils/common/file.ts` |
| 通用导入弹窗 | `star-pivot-ui/src/components/core/forms/art-import-dialog/index.vue` |
| 参考示例 | `star-pivot-ui/src/views/system/user/index.vue` |

---

## 二、接口清单

所有接口前缀：`/api/common/import-export`（具体以项目配置的 baseURL 为准）。

| 功能 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 导入 | POST | `/import/{businessType}` | 提交 Excel 解析后的行数据 + 导入选项 |
| 导出 | POST | `/export/{businessType}` | 按查询条件导出，返回 Excel 文件流 |
| 下载模板 | GET | `/template/{businessType}` | 下载导入模板 Excel |

`businessType`：业务类型标识，如 `user`、`dept`、`role` 等，与各业务实现的 `getBusinessType()` 保持一致。

---

## 三、后端对接

### 3.1 实现 `ImportExportService`

在对应业务的 Service 上实现 `ImportExportService`，并实现四个方法：

```java
public interface ImportExportService {
    String getBusinessType();                                    // 业务类型，如 "user"
    ImportExportResult importData(List<Map<String, Object>> rowList, Map<String, Object> options);
    List<Map<String, Object>> exportData(Map<String, Object> queryParams);
    ImportTemplate getImportTemplate();
}
```

| 方法 | 作用 |
|------|------|
| `getBusinessType()` | 返回唯一业务类型，如 `"user"`，用于路由到该实现 |
| `importData(rowList, options)` | 按行导入；`rowList` 的 key 为表头名，value 为单元格值；`options` 见下文 |
| `exportData(queryParams)` | 按查询条件查出数据，转换为 `List<Map<String, Object>>`，**Map 的 key 即导出表头** |
| `getImportTemplate()` | 返回 `ImportTemplate`：`headers` 表头列表 + `sampleData` 示例行 |

### 3.2 `ImportExportResult` 与 `ImportTemplate`

- **ImportExportResult**：  
  - `successCount`、`failCount`、`errorMessages`  
  - 每行导入失败时，建议往 `errorMessages` 里追加简要说明（如「第 N 行：xxx」）。

- **ImportTemplate**：  
  - `headers`：`List<String>`，表头顺序与 Excel 列顺序一致；  
  - `sampleData`：`List<Map<String, Object>>`，key 与 `headers` 对应，用于生成带示例的模板。

### 3.3 导入选项 `options`

通用约定（具体由各业务在 `importData` 内解析）：

| key | 类型 | 说明 |
|-----|------|------|
| `overwrite` | Boolean | 是否更新已存在数据。如用户模块：`true` 时按用户名更新，否则仅新增 |

可在本接口内扩展更多业务相关选项。

### 3.4 注册到工厂

实现类加 `@Service` 等 Spring 组件注解即可。`ImportExportServiceFactory` 会收集所有 `ImportExportService` 实现，并以 `getBusinessType()` 为 key 注册，**无需额外配置**。

### 3.5 用户模块示例（摘要）

- `SysUserService` 实现 `ImportExportService`，`getBusinessType()` 返回 `"user"`。
- `importData`：根据 `options.overwrite` 决定新增还是按用户名更新；逐行校验、落库，并填充 `ImportExportResult`。
- `exportData`：将 `UserReqBo` 等查询参数转为 `queryParams`，查用户列表，再转为 `List<Map<String, Object>>`，key 如「用户账号」「用户昵称」等。
- `getImportTemplate`：表头与导出一致，并附带一行示例数据。

详细逻辑见 `SysUserServiceImpl`。

## 四、前端对接

### 4.0 通用导入弹窗组件（推荐）

**`ArtImportDialog`**（`components/core/forms/art-import-dialog/index.vue`）已封装完整导入流程：Excel 上传、解析、模板下载、可选「覆盖已存在数据」、调用通用导入接口、成功/失败提示。其他页面可直接使用，无需再写解析与接口调用逻辑。

**使用示例：**

```vue
<template>
  <ElButton @click="importVisible = true">导入 xxx</ElButton>
  <ArtImportDialog
    v-model="importVisible"
    business-type="user"
    title="用户导入"
    :show-overwrite="true"
    overwrite-label="是否更新已经存在的用户数据"
    @success="refreshData"
  />
</template>

<script setup lang="ts">
  import ArtImportDialog from '@/components/core/forms/art-import-dialog/index.vue'

  const importVisible = ref(false)
  // refreshData 为列表刷新方法，导入成功后调用
</script>
```

**Props：**

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `modelValue` | `boolean` | `false` | 弹窗显隐，支持 `v-model` |
| `businessType` | `string` | 必填 | 业务类型，如 `user`、`dept`、`role` |
| `title` | `string` | `'数据导入'` | 弹窗标题 |
| `showOverwrite` | `boolean` | `false` | 是否显示「覆盖已存在数据」复选框 |
| `overwriteLabel` | `string` | `'是否更新已经存在的数据'` | 覆盖选项文案 |
| `closeOnSuccess` | `boolean` | `true` | 导入成功后是否自动关闭弹窗 |

**事件：**

| 事件 | 参数 | 说明 |
|------|------|------|
| `success` | `{ successCount, failCount, errorMessages? }` | 导入完成（含部分失败）时触发，可在此刷新列表 |
| `error` | `err` | 解析或接口报错时触发 |

**插槽：** `extra-options` — 在「下载模板」上方插入自定义表单项（如更多业务选项）。

若不使用该组件，可自行实现弹窗并调用下方 API；若使用，**可跳过 4.2、4.4**，仅保留导出与列表刷新逻辑。

### 4.1 API 封装（`api/common/import-export.ts`）

```ts
import {
  fetchImportData,
  fetchExportData,
  fetchDownloadTemplate,
} from '@/api/common/import-export'
```

| 方法 | 用途 |
|------|------|
| `fetchImportData(businessType, data, options?)` | 导入；`data` 为 Excel 解析得到的行数组 |
| `fetchExportData(businessType, queryParams?, filename?)` | 导出；触发下载，可选 `filename` |
| `fetchDownloadTemplate(businessType, filename?)` | 下载模板；触发下载 |

`ArtImportDialog` 内部已使用 `fetchImportData`、`fetchDownloadTemplate`，页面只需按需使用 `fetchExportData`。

### 4.2 导入流程（含 Excel 解析，未用组件时）

1. 用户选择 Excel（.xls / .xlsx），前端用 **XLSX** 解析为 `Array<Record<string, unknown>>`，每个元素为一行的 key-value（key = 表头）。
2. 组装 `options`（如 `{ overwrite: true }`），调用  
   `fetchImportData('user', rows, options)`（这里 `user` 换成你的 `businessType`）。
3. 根据返回的 `successCount`、`failCount`、`errorMessages` 做成功/失败提示，必要时刷新列表。

用户模块中的解析示例（`index.vue`）：

```ts
import * as XLSX from 'xlsx'

function parseExcelFile(file: File): Promise<Array<Record<string, unknown>>> {
  return new Promise((resolve, reject) => {
    const reader = new FileReader()
    reader.onload = (e) => {
      try {
        const workbook = XLSX.read(e.target?.result, { type: 'array' })
        const sheet = workbook.Sheets[workbook.SheetNames[0]]
        const rows = XLSX.utils.sheet_to_json(sheet) as Array<Record<string, unknown>>
        resolve(rows)
      } catch (err) {
        reject(err)
      }
    }
    reader.onerror = () => reject(reader.error)
    reader.readAsArrayBuffer(file)
  })
}
```

### 4.3 导出流程

1. 如有搜索表单，将表单值作为 `queryParams`（与列表查询参数一致即可）。
2. 调用 `fetchExportData(businessType, queryParams)`，接口返回 Excel 流，前端会触发下载并提示成功/失败。

示例：

```ts
const handleExport = async () => {
  try {
    await fetchExportData('user', { ...searchForm.value })
  } catch (e) {
    console.error('导出失败', e)
  }
}
```

### 4.4 下载模板

```ts
const handleDownloadTemplate = async () => {
  try {
    await fetchDownloadTemplate('user')  // 替换为你的 businessType
  } catch (e) {
    console.error('下载模板失败', e)
  }
}
```

### 4.5 页面 UI 建议

- **导入（推荐）**：使用 `ArtImportDialog`，按钮打开弹窗，`@success` 中刷新列表。详见 **4.0**。
- **导入（自定义）**：按钮 → 弹窗（ElUpload + 选择文件）→ 确认时解析 Excel → 调用 `fetchImportData`；若有「是否更新已存在数据」，用 `ElCheckbox` 绑定 `overwrite`，一并传入 `options`。
- **导出**：按钮 → 直接调用 `fetchExportData(businessType, queryParams)`。
- **模板**：`ArtImportDialog` 已内置「下载模板」链接；自定义弹窗时在导入区域内提供链接，调用 `fetchDownloadTemplate(businessType)`。

参考示例：`views/system/user/index.vue`（使用 `ArtImportDialog` + 导出按钮）。

## 五、Excel 格式约定

### 5.1 表头与数据 key

- 导入：Excel **第一行为表头**，解析后每行的 key 即为表头字符串（如「用户账号」「用户昵称」）。  
  后端 `importData` 中按这些 key 取值，**需与模板、导出表头完全一致**。
- 导出：`exportData` 返回的 `List<Map<String, Object>>` 中，**Map 的 key 即导出表头**，顺序按遍历顺序即可（建议使用 `LinkedHashMap` 保序）。
- 模板：`getImportTemplate()` 的 `headers` 与 `sampleData` 的 key 一致，且与导入/导出表头一致。

### 5.2 文件格式

- 支持 `.xls`、`.xlsx`。  
- 模板与导出的文件由后端 EasyExcel 生成，格式统一。

## 六、常见问题

1. **404 / 未找到业务类型**  
   - 检查该业务的 Service 是否实现 `ImportExportService` 且被 Spring 扫描到。  
   - 确认 `getBusinessType()` 与前端、接口路径中的 `businessType` 一致（如 `user`）。

2. **导入后数据错列**  
   - 表头名必须与后端 `importData`、`getImportTemplate` 中使用的 key 完全一致（含空格、中英文）。  
   - 模板表头与 Excel 实际表头一致。

3. **导出文件乱码或无法打开**  
   - 前端请求需带 `responseType: 'blob'`，已有封装可直接用 `fetchExportData`。  
   - 后端设置 `Content-Disposition`、`Content-Type` 正确，已由通用 Controller 处理。

4. **覆盖更新不生效**  
   - 确认前端把 `overwrite: true` 放入 `options` 并随 `fetchImportData` 提交。  
   - 后端 `importData` 中正确读取 `options.get("overwrite")` 并分支逻辑。

## 七、对接检查清单

**后端**

- [ ] 实现 `ImportExportService`，实现四方法。  
- [ ] `getBusinessType()` 与接口路径、前端使用一致。  
- [ ] `exportData` 返回的 `Map` key 与模板、导入表头一致。  
- [ ] `getImportTemplate` 的 `headers`、`sampleData` 与导入/导出一致。  
- [ ] 如需覆盖更新，在 `importData` 中解析 `options.overwrite` 并实现更新逻辑。

**前端**

- [ ] 使用 `fetchImportData` / `fetchExportData` / `fetchDownloadTemplate`，并传入正确的 `businessType`。  
- [ ] 导入前用 XLSX 解析 Excel，表头与后端约定一致。  
- [ ] 导出时传入与列表查询一致的 `queryParams`（可选）。  
- [ ] 按 `ImportExportResult` 做成功/失败提示，并在导入成功后刷新列表。

按上述步骤对接即可在其他页面复用同一套导入、导出、模板能力。若业务有特殊约定，可在本说明基础上补充到 `doc/` 或项目 README。
