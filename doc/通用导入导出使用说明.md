# 通用导入导出使用说明

本文档说明如何在 StarPivot 中对接 **EasyExcel 模块化导入导出**能力。各业务在自身 Controller 暴露 `/export`、`/import`、`/importTemplate`，通过框架 `ExcelToolkit` + `ExcelBizHandler` 复用同一套读写与 HTTP 封装。

> **说明**：旧版「统一入口 `POST /api/common/import-export/{businessType}` + 前端 XLSX 解析 + `ImportExportService`」已移除。新接入请完全按本文档；`api/common/import-export.ts` 仅保留废弃提示，勿再使用。

---

## 一、架构概览

```
┌─────────────────┐     multipart      ┌──────────────────┐
│ ExcelImportDialog│ ────────────────► │ Controller       │
│ + fetchExcel*   │     blob 下载      │ /export          │
└─────────────────┘                    │ /import          │
        ▲                              │ /importTemplate  │
        │                              └────────┬─────────┘
        │ 业务 API 封装                            │
        │                              ┌────────▼─────────┐
        │                              │ ExcelToolkit     │
        │                              │ + ExcelBizHandler│
        │                              └────────┬─────────┘
        │                                       │
        │                              ┌────────▼─────────┐
        └──────────────────────────────│ Service 落库逻辑 │
                                       │ *Excel 行类型    │
                                       └──────────────────┘
```

| 层级 | 职责 |
|------|------|
| **`*Excel` 行类** | `@ExcelProperty` 定义表头与列顺序，导入/导出/模板共用 |
| **`ExcelBizHandler`** | 委托 Service：查导出数据、执行导入、定制文件名/示例行 |
| **`ExcelToolkit`** | EasyExcel 读写、文件流响应、`importFile` 统一入口 |
| **Controller** | 三个端点 + 权限 + `@Log`；导出/模板加 `@NoResponseWrapper` |
| **前端 `excel.ts`** | `fetchExcelExport` / `fetchExcelImport` / `fetchExcelTemplate` |
| **`ExcelImportDialog`** | 上传、覆盖选项、模板下载、结果提示 |

### 相关文件

| 类型 | 路径 |
|------|------|
| 工具类 | `star-pivot-framework/.../excel/ExcelToolkit.java` |
| 处理器接口 | `star-pivot-framework/.../excel/ExcelBizHandler.java` |
| 导入选项 | `star-pivot-framework/.../excel/ExcelImportOptions.java` |
| 导入结果 | `star-pivot-framework/.../excel/ExcelImportResult.java` |
| 前端 API | `star-pivot-ui/src/api/common/excel.ts` |
| 导入弹窗 | `star-pivot-ui/src/components/core/forms/excel-import-dialog/index.vue` |
| Blob 下载 | `star-pivot-ui/src/utils/common/file.ts` |

|------|---------|--------|------------|
| 用户 | `SysUserExcelHandler` | `SysUserExcel` | `SysUserController` |

## 二、接口约定

每个业务模块在**自身**路由下提供三个接口（路径以模块 `@RequestMapping` 为准）。

| 功能 | 方法 | 路径 | 请求 | 响应 |
|------|------|------|------|------|
| 导出 | POST | `/export` | JSON 查询体（与列表查询一致） | Excel 文件流（`ResponseEntity<byte[]>`） |
| 导入 | POST | `/import` | `multipart/form-data`：`file` 必填；`updateSupport` 可选 | `Result<ExcelImportResult>` |
| 下载模板 | GET | `/importTemplate` | 查询参数（按业务需要） | Excel 文件流 |

### 通用表单字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `file` | `MultipartFile` / `File` | Excel 文件，`.xls` / `.xlsx` |
| `updateSupport` | `boolean` | 是否覆盖更新已存在数据；默认 `false`。前端勾选后传 `true` |

业务扩展参数（如商品属性 `attrType`）在 Controller 用 `@RequestParam` 接收，通过 `ExcelImportOptions.builder().param(...)` 传给 Handler。

### 注解要求

- **导出、模板**：方法加 `@NoResponseWrapper`，避免全局包装破坏文件流。
- **权限**：`@PreAuthorize` 与菜单 `perms` 一致，命名规则见 [`权限编码与数据权限规范.md`](权限编码与数据权限规范.md)（如 `system:user:export`、`mall:group:import`）。
- **操作日志**：`@Log(..., businessType = EXPORT / IMPORT)`。

## 三、后端对接（分步）

### 3.1 定义 Excel 行类 `XxxExcel`

在模块 `domain/excel/` 下新建类，使用 EasyExcel 注解。**导入、导出、模板三处共用此类**。

```java
@Data
public class SysUserExcel {

    @ExcelProperty(value = "用户账号", index = 0)
    private String userName;

    @ExcelProperty(value = "用户昵称", index = 1)
    private String nickName;
    // ...
}
```

注意：

- `value` 为表头中文（或业务约定文案），`index` 从 0 起连续编号。
- 需要「按 ID 覆盖更新」时，模板中应包含 ID 列（见商品属性 `属性ID`）。
- 枚举/状态建议用可读文案（如「正常」「停用」），在 Service 中转换。

### 3.2 实现 `ExcelBizHandler<T, Q>`

```java
@Component
@RequiredArgsConstructor
public class SysUserExcelHandler implements ExcelBizHandler<SysUserExcel, UserReqBo> {

    private final SysUserService sysUserService;

    @Override
    public List<SysUserExcel> listForExport(UserReqBo query) {
        return sysUserService.listForExport(query);
    }

    @Override
    public ExcelImportResult importRows(List<SysUserExcel> rows, ExcelImportOptions options) {
        return sysUserService.importFromExcel(rows, options.isUpdateSupport());
    }

  // 以下为可选覆盖
    @Override
    public List<SysUserExcel> templateSampleRows(UserReqBo query) { /* 示例行 */ }

    @Override
    public String sheetName(UserReqBo query) { return "用户"; }

    @Override
    public String exportFileName(UserReqBo query) {
        return "user_export_" + System.currentTimeMillis() + ".xlsx";
    }

    @Override
    public String templateFileName(UserReqBo query) {
        return "user_import_template.xlsx";
    }
}
```

| 方法 | 说明 |
|------|------|
| `listForExport(Q)` | 按查询条件返回导出行；无数据时 Toolkit 返回「没有可导出的数据」 |
| `importRows(List<T>, ExcelImportOptions)` | 逐行校验落库，返回 `ExcelImportResult` |
| `templateSampleRows(Q)` | 默认空列表；有值则模板带示例行 |
| `sheetName` / `exportFileName` / `templateFileName` | 工作表名与下载文件名 |

### 3.3 Service 层

- **导出**：查询实体后映射为 `List<XxxExcel>`。
- **导入**：推荐逐行 `try/catch`，失败时 `result.addError("第 N 行：...")` 并 `failCount++`，成功则 `successCount++`（用户模块即此模式）。
- 也可在全部校验通过后批量落库，使用 `ExcelImportResult.allSuccess(count)`（商品属性模块）。

`ExcelImportOptions` 除 `updateSupport` 外，可用 Builder 传业务参数：

```java
ExcelImportOptions options = ExcelImportOptions.builder()
        .updateSupport(updateSupport)
        .param("attrType", attrType)
        .build();
// Handler 内：options.getIntParam("attrType")
```

### 3.4 Controller 薄封装

```java
/** 导出 */
@Log(title = "用户管理", businessType = AppConstants.BusinessType.EXPORT)
@PreAuthorize("hasAuthority('system:user:export')")
@NoResponseWrapper
@PostMapping("/export")
public ResponseEntity<?> exportUsers(@RequestBody(required = false) UserReqBo userReqBo) {
    return ExcelToolkit.export(
            sysUserExcelHandler,
            userReqBo != null ? userReqBo : new UserReqBo(),
            SysUserExcel.class);
}

/** 导入 */
@Log(title = "用户管理", businessType = AppConstants.BusinessType.IMPORT)
@PreAuthorize("hasAuthority('system:user:import')")
@PostMapping("/import")
public Result<ExcelImportResult> importUsers(
        @RequestParam("file") MultipartFile file,
        @RequestParam(value = "updateSupport", defaultValue = "false") boolean updateSupport)
        throws Exception {
    return ExcelToolkit.importFile(
            file, sysUserExcelHandler, ExcelImportOptions.of(updateSupport), SysUserExcel.class);
}

/** 模板 */
@PreAuthorize("hasAuthority('system:user:import')")
@NoResponseWrapper
@GetMapping("/importTemplate")
public ResponseEntity<?> importTemplate() {
    return ExcelToolkit.downloadTemplate(sysUserExcelHandler, new UserReqBo(), SysUserExcel.class);
}
```

带业务参数的导入（商品属性示例）：

```java
@PostMapping("/import")
public Result<?> importData(
        @RequestParam("file") MultipartFile file,
        @RequestParam("attrType") Integer attrType,
        @RequestParam(value = "updateSupport", defaultValue = "false") boolean updateSupport)
        throws Exception {
    ExcelImportOptions options = ExcelImportOptions.builder()
            .updateSupport(updateSupport)
            .param("attrType", attrType)
            .build();
    return ExcelToolkit.importFile(file, pmsAttrExcelHandler, options, PmsAttrExcel.class);
}
```

## 四、前端对接

### 4.1 业务 API 封装（推荐）

在 `src/api/<模块>/` 中封装三个方法，内部调用 `api/common/excel.ts`：

```ts
import { fetchExcelExport, fetchExcelImport, fetchExcelTemplate } from '@/api/common/excel'

export function fetchExportUser(params: Api.SystemManage.UserSearchParams) {
  return fetchExcelExport({
    url: '/api/sys/user/export',
    data: params as Record<string, unknown>,
    filenameFallback: `user_export_${Date.now()}.xlsx`,
    successMessage: false  // 页面自行提示时可关闭
  })
}

export function fetchImportUserExcel(file: File, updateSupport = false) {
  return fetchExcelImport<ExcelImportResultVo>({
    url: '/api/sys/user/import',
    file,
    updateSupport
  })
}

export function fetchDownloadUserImportTemplate() {
  return fetchExcelTemplate({
    url: '/api/sys/user/importTemplate',
    filenameFallback: 'user_import_template.xlsx',
    successMessage: false
  })
}
```

带扩展表单字段（与后端 `@RequestParam` 对应）：

```ts
export function fetchImportAttr(file: File, attrType: 0 | 1, updateSupport = false) {
  return fetchExcelImport({
    url: '/api/mall/attr/import',
    file,
    updateSupport,
    extraFormData: { attrType }
  })
}
```

| 方法 | 说明 |
|------|------|
| `fetchExcelExport` | `responseType: 'blob'`，解析 `Content-Disposition` 触发下载 |
| `fetchExcelImport` | `FormData` 上传；返回 `data` 含 `successCount` / `failCount` / `errorMessages` |
| `fetchExcelTemplate` | GET + blob 下载模板 |

### 4.2 使用 `ExcelImportDialog`（推荐）

组件路径：`components/core/forms/excel-import-dialog/index.vue`。

**不再**由组件感知 `businessType`；通过函数 props 注入本业务的导入与模板 API。

```vue
<ElButton @click="importDialogVisible = true">导入</ElButton>

<ExcelImportDialog
  v-model="importDialogVisible"
  title="用户导入"
  :show-overwrite="true"
  overwrite-label="是否更新已经存在的用户数据"
  :import-fn="doImportUser"
  :download-template-fn="fetchDownloadUserImportTemplate"
  @success="refreshData"
/>
```

```ts
const importDialogVisible = ref(false)

const doImportUser = (file: File, updateSupport: boolean) =>
  fetchImportUserExcel(file, updateSupport)
```

**Props：**

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `modelValue` | `boolean` | `false` | 弹窗显隐，`v-model` |
| `title` | `string` | `'数据导入'` | 标题 |
| `showOverwrite` | `boolean` | `true` | 是否显示覆盖复选框 |
| `overwriteLabel` | `string` | `'是否更新已存在的数据（需填写ID列）'` | 覆盖选项文案 |
| `importFn` | `(file, updateSupport) => Promise` | 必填 | 调用业务 `fetchExcelImport` |
| `downloadTemplateFn` | `() => Promise<void>` | 必填 | 调用业务 `fetchExcelTemplate` |

**事件：**

| 事件 | 说明 |
|------|------|
| `success` | 导入流程结束（含部分失败）后触发，可刷新列表 |
| `error` | 模板下载或导入请求异常 |

**插槽：** `extra-options` — 在覆盖选项下方插入自定义表单项（若选项需在导入前由用户选择，应同步到 `importFn` 闭包或 `extraFormData`）。

组件已处理：文件校验、成功/部分失败提示、模板下载 loading。**无需**再引入 `xlsx` 库做前端解析。

### 4.3 导出按钮

```ts
const handleExport = async () => {
  try {
    await fetchExportUser({ ...searchForm.value })
    ElMessage.success('导出成功')
  } catch (e) {
    ElMessage.error('导出失败')
  }
}
```

`queryParams` 与列表查询保持一致；无数据时后端返回 JSON 错误，前端按异常处理即可。

### 4.4 权限与按钮

与现有页面一致，使用 `v-auth="'module:resource:import'"` 等控制导入/导出按钮显隐。

## 五、Excel 与数据约定

1. **表头以 `@ExcelProperty(value)` 为准**，与 Excel 第一行一致；修改表头需同步改行类与 Service 映射逻辑。
2. **列顺序以 `index` 为准**，导出与模板列顺序与行类一致。
3. **文件格式**：`.xls`、`.xlsx`；由后端 EasyExcel 生成，前端直接上传原始文件。
4. **覆盖更新**：`updateSupport=true` 时，业务需根据唯一键或 ID 列执行更新；模板应包含必要主键列。
5. **部分失败**：`failCount > 0` 时 `Result` 仍为成功态，但 `data.errorMessages` 含明细；`ExcelImportDialog` 会以 `warning` 展示前 5 条。

## 七、常见问题

**1. 导出文件打不开或只有几 KB 的 JSON**

- 确认导出接口有 `@NoResponseWrapper`。
- 前端必须使用 `fetchExcelExport`（`responseType: 'blob'`），不要用普通 JSON 请求。

**2. 导入报「请上传 Excel 文件」**

- 确认 `FormData` 字段名为 `file`，且 `Content-Type` 为 `multipart/form-data`（`fetchExcelImport` 已处理）。

**3. 导入后列错位或字段全空**

- 检查 Excel 表头是否与 `@ExcelProperty(value)` 完全一致（含空格、标点）。
- 检查 `index` 是否与模板列顺序一致。

**4. 勾选覆盖但不更新**

- 前端是否传入 `updateSupport: true`（`ExcelImportDialog` 的覆盖复选框）。
- 后端 `importRows` 是否使用 `options.isUpdateSupport()`，且模板是否包含用于匹配的主键/唯一键列。

**5. 扩展参数未生效**

- 后端：`ExcelImportOptions.builder().param("key", value)`。
- 前端：`extraFormData: { key: value }`，字段名与 `@RequestParam` 一致。

**6. 仍看到 `ArtImportDialog` / `import-export.ts`**

- 二者为旧方案遗留；新功能请使用 `ExcelImportDialog` + `excel.ts`。

## 八、与旧方案差异（迁移参考）

| 项目 | 旧方案 | 当前方案 |
|------|--------|----------|
| 入口 | 统一 `/api/common/import-export/*` | 各模块 `/export` `/import` `/importTemplate` |
| 后端扩展 | `ImportExportService` + `Map` 行数据 | `ExcelBizHandler` + 强类型 `*Excel` |
| 解析位置 | 前端 XLSX → JSON 提交 | 后端 EasyExcel 解析 `MultipartFile` |
| 覆盖字段 | `options.overwrite` | `updateSupport`（表单 + `ExcelImportOptions`） |
| 前端组件 | `ArtImportDialog` + `businessType` | `ExcelImportDialog` + `import-fn` |

按本文档对接后，新页面可与用户管理、商品属性、属性分组等模块保持同一套体验与维护方式。
