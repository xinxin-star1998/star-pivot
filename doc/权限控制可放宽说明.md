# 权限控制可放宽说明

## 一、已使用或建议改为 `isAuthenticated()` 的接口

这些接口**仅服务当前登录用户自身**（如获取本人菜单、本人信息、本人路由），不涉及管理他人数据，因此「已登录即可访问」足够，无需 `hasAuthority` 等更细粒度权限。

| 接口 | 当前权限 | 建议 | 说明 |
|------|----------|------|------|
| `GET /auth/userinfo` | 无 `@PreAuthorize`（仅认证） | 保持 | 获取当前用户信息，已有认证即可 |
| `GET /sys/menu/userMenuTree` | `isAuthenticated()` | 保持 | 获取当前用户菜单树，已放宽 |
| `DELETE /avatar/delete` | `isAuthenticated()` | 保持 | 删除本人头像，内部再校验 userId |
| `GET /router/dynamic-routes` | `hasAuthority('system:menu:query')` | **改为 `isAuthenticated()`** | 按当前用户返回动态路由，与 userMenuTree 类似 |

---

## 二、`/router/dynamic-routes` 建议修改原因

- 实现逻辑：通过 `SecurityContextUtils.getUserId()` 取当前用户，再调用 `getUserMenuTree(userId)`，**只返回当前用户的路由**。
- 与 `getUserMenuTree` 相同，属于**登录后构建前台所需的基础数据**，任何已登录用户都应可访问。
- 若保留 `system:menu:query`，未配置该权限的用户无法拉取动态路由，可能影响正常使用。

因此建议将 `getDynamicRoutes` 的 `@PreAuthorize` 改为 `isAuthenticated()`。

---

## 三、其他接口保持现有权限的原因

以下均为**管理类或敏感操作**，继续使用 `hasAuthority` / `hasRole` 等严格校验：

| 类型 | 示例 | 说明 |
|------|------|------|
| 菜单管理 | `menuTree`、`getParent`、`getById`、`add`、`edit`、`remove` | 管理全局菜单，需 `system:menu:*` |
| 用户管理 | `pageList`、`getUserById`、`add`、`update`、`resetPwd`、`changeStatus`、`remove` | 管理用户及重置密码等，需 `system:user:*` |
| 角色管理 | `list`、`getInfo`、`allocatedList`、`unallocatedList`、`assignUser` 等 | 角色与权限分配，需 `system:role:*` |
| 部门 / 岗位 | `tree`、`getInfo`、`add`、`edit`、`remove` | 组织架构管理，需 `system:dept:*`、`system:post:*` |
| 字典 | 类型 / 数据的增删改查 | 系统字典管理，需 `system:type:*`、`system:data:*` |
| 日志 | 操作日志、登录日志的查询与清理 | 运维审计，需 `system:operlog:*`、`system:logininfor:*` |
| 代码生成 | GenController 各接口 | 代码生成工具，需 `tool:gen:*` |

---

## 四、可选后续优化（按需评估）

1. **字典只读接口**  
   若字典（如下拉选项）在多个业务页使用，且不希望给所有账号配置 `system:type:query` / `system:data:query`，可考虑：
   - 新增仅做查询的接口（如 `GET /dict/data/type/{dictType}` 只读），使用 `isAuthenticated()`；或  
   - 将现有字典查询接口的权限改为 `isAuthenticated()`，需评估是否与「字典管理」菜单权限一致。

2. **头像上传**  
   `AvatarController` 上传未加 `@PreAuthorize`，在方法内通过 `hasPermissionToOperateUser` 校验「本人或管理员」。若希望统一用注解表达「需登录」，可增加 `@PreAuthorize("isAuthenticated()")`，不改现有业务逻辑。

---

## 五、本次实际修改

- **RouterController.getDynamicRoutes**：`@PreAuthorize("hasAuthority('system:menu:query')")` → `@PreAuthorize("isAuthenticated()")`。

其他接口保持现状；若后续有类似「仅当前用户」的接口，可优先考虑 `isAuthenticated()`。
