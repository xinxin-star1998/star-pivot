# StarPivot 项目依赖引用梳理

本文档描述仓库 **Maven 多模块** 与 **前端独立工程** 的依赖关系，便于新增模块、排查类冲突与理解分层约束。

相关文档：[`架构图与流程图.md`](架构图与流程图.md)、[`star-pivot-security-使用说明.md`](star-pivot-security-使用说明.md)、[`权限编码与数据权限规范.md`](权限编码与数据权限规范.md)。

---

## 一、项目结构总览

```
StarPivot（根 POM，packaging=pom）
├── star-pivot-dependencies          # BOM：统一版本，无源码
├── star-pivot-framework             # 框架层聚合（packaging=pom）
│   ├── star-pivot-framework-core    # 底座：domain / 异常 / 工具 / EasyExcel 等
│   ├── star-pivot-framework-log     # 操作日志 @Log
│   ├── star-pivot-framework-file    # MinIO / OSS 文件存储
│   ├── star-pivot-framework-web     # Web 层抽象（LoginUserInfo 等）
│   ├── star-pivot-security          # JWT、SecurityConfig、黑名单
│   ├── star-pivot-framework-datasource  # 多数据源（可选能力）
│   └── star-pivot-framework-boot-starter  # 应用层一站引入（推荐唯一入口）
├── star-pivot-module                # 业务模块聚合（packaging=pom）
│   ├── star-pivot-dict              # 字典（被 system 依赖）
│   ├── star-pivot-system            # 系统管理（用户/角色/菜单/认证等）
│   ├── star-pivot-generator         # 代码生成（已解耦 system）
│   ├── star-pivot-quartz            # 定时任务（已解耦 system）
│   ├── star-pivot-monitor           # 监控（依赖 system）
│   └── star-pivot-mall              # 商城业务
├── star-pivot-controller            # 可执行 JAR：Controller + 启动类
└── star-pivot-ui                    # 前端 Vue3 + Vite（独立 npm/pnpm 体系）
```

### 分层原则

| 规则 | 说明 |
|------|------|
| 构建顺序 | `dependencies` → `framework` → `module` → `controller` |
| 禁止反向依赖 | 框架层不得依赖业务模块；业务模块不得依赖 `controller` |
| 框架接入 | 业务模块优先只依赖 **`star-pivot-framework-boot-starter`**，避免直接引用 `framework-core` / `security` 子模块（除非有明确理由） |
| 业务解耦 | `generator`、`quartz` 不再依赖 `system`；跨业务能力通过 `controller` 聚合或接口下沉 |
| 版本管理 | 子模块依赖不写 `<version>`，由 BOM `star-pivot-dependencies` 统一管理 |
| 历史说明 | 已移除独立模块 `star-pivot-common`，能力并入 `framework-core`（包名 `com.star.pivot.framework`） |

## 二、根 POM 与 BOM

### 2.1 根 POM（`StarPivot/pom.xml`）

- `packaging=pom`，聚合四个子工程：`dependencies`、`framework`、`module`、`controller`。
- 通过 `<dependencyManagement>` **import** `star-pivot-dependencies`，自身不声明业务依赖。
- JDK **17**；Spring Boot **3.2.10**（构建插件版本在根 POM `properties` 中）。

### 2.2 star-pivot-dependencies（BOM）

- 仅 `<dependencyManagement>` + `<properties>`，不参与编译。
- **第三方**：Spring Boot BOM、MyBatis-Plus、MySQL、Druid、JWT（jjwt）、Hutool、POI/EasyExcel、MinIO、Quartz、SpringDoc、ArchUnit 等。
- **内部模块**：所有 `star-pivot-*` artifact 版本 `${star-pivot.version}`。

首次或清理后构建需先安装 BOM（见第七节）。

## 三、依赖关系图

### 3.1 框架层（自底向上）

```
star-pivot-framework-core
        ▲
        │ 仅依赖 core
   ┌────┴────┬────────────┐
   │         │            │
  log      file          web
   └────┬────┴────────────┘
        ▼
star-pivot-security  （core + log + file + web + Spring Security + Redis + JWT）
        │
        ├──────────────────────────────────┐
        ▼                                  ▼
star-pivot-framework-datasource    star-pivot-framework-boot-starter
（多数据源，独立）                    （security + datasource
                                      + mybatis-plus + springdoc + redis）
```

### 3.2 业务层 → 应用层

```
                    ┌─────────────────────────┐
                    │  star-pivot-controller   │
                    └───────────┬─────────────┘
                                │
     ┌──────────┬───────────┬───┴───┬──────────┬──────────┐
     ▼          ▼           ▼       ▼          ▼          ▼
  system      dict        mall  generator   quartz    monitor
     │          │           │       │          │          │
     │          │           │       │          │          └──► system
     └──► dict  │           │       │          │
                └───────────┴───────┴──────────┴──► boot-starter（均依赖）
```

**说明**：`controller` 不直接依赖 `framework` 子模块，仅依赖 `boot-starter` 与各业务 JAR；`monitor` 因需调用用户/部门等服务而依赖 `system`。

---

### 4.7 star-pivot-framework-boot-starter（推荐入口）

| 项 | 内容 |
|----|------|
| 职责 | 业务模块与 controller **统一**引入的框架能力包 |
| 内部依赖 | `star-pivot-security`（传递 core/log/file/web）、`framework-datasource` |
| 额外第三方 | mybatis-plus-boot-starter、springdoc-openapi、spring-boot-starter-data-redis |

**约定**：新建业务模块只需：

```xml
<dependency>
    <groupId>com.star.pivot</groupId>
    <artifactId>star-pivot-framework-boot-starter</artifactId>
</dependency>
```

即可得到安全、Redis、MyBatis-Plus、OpenAPI 等基础能力（仍须在 controller 配置数据源、Redis 连接等）。

## 五、业务模块间依赖矩阵

| 模块 ↓ 依赖 → | dict | system | boot-starter | security |
|---------------|------|--------|----------------|----------|
| dict | — | — | ✓ | ✓ |
| system | ✓ | — | ✓ | （传递） |
| generator | — | — | ✓ | （传递） |
| quartz | — | — | ✓ | （传递） |
| monitor | — | ✓ | ✓ | （传递） |
| mall | — | — | ✓ | ✓ |
| controller | ✓ | ✓ | ✓ | （传递） |

**原则**：业务模块之间尽量避免横向依赖；当前仅 `system→dict`、`monitor→system` 为显式业务依赖。

## 八、构建与运行

### 8.1 全量构建

```bash
# 推荐：在仓库根目录（StarPivot/）执行，-am 会自动构建依赖模块
mvn clean install -DskipTests
```

若 BOM 未安装到本地仓库，可先：

```bash
mvn install -f star-pivot-dependencies/pom.xml
mvn clean install -DskipTests
```

### 8.2 仅启动后端

```bash
mvn spring-boot:run -pl star-pivot-controller -am
```

### 8.3 仅编译某一业务模块

```bash
mvn compile -pl star-pivot-mall -am
```

### 8.4 前端

```bash
cd star-pivot-ui
pnpm install
pnpm dev
```

## 九、新增业务模块指引

1. 在 `star-pivot-module/pom.xml` 的 `<modules>` 中增加子模块。
2. 子模块 `pom.xml` 父级指向 `star-pivot-module`，依赖 **`star-pivot-framework-boot-starter`** + `mybatis-plus` 等按需第三方。
3. 在 `star-pivot-dependencies/pom.xml` 的 `<dependencyManagement>` 中登记新 artifact（与现有内部模块一致）。
4. 在 `star-pivot-controller/pom.xml` 中引入新业务 JAR。
5. 在 `star-pivot-controller` 下新增 `com.star.pivot.controller.*` 包及启动扫描（若包名在默认扫描路径内则无需改启动类）。
6. **不要**让 framework 依赖新业务模块；跨模块调用优先评估是否应下沉到 `system` 或抽接口。

## 十、依赖流向小结

| 层级 | 模块 | 直接依赖 |
|------|------|----------|
| BOM | star-pivot-dependencies | （无，被 import） |
| 底座 | framework-core | 仅第三方 |
| 框架能力 | log / file / web | framework-core |
| 安全 | star-pivot-security | core + log + file + web |
| 数据源 | framework-datasource | 第三方 JDBC/AOP |
| 启动聚合 | framework-boot-starter | security + datasource + MP + springdoc + redis |
| 业务 | dict | boot-starter + security |
| 业务 | system | dict + boot-starter |
| 业务 | generator / quartz | boot-starter |
| 业务 | monitor | boot-starter + system |
| 业务 | mall | boot-starter + security |
| 应用 | controller | boot-starter + 全部业务模块 + MySQL/Druid/Actuator 等 |

## 十一、常见问题

**1. 业务模块能否只依赖 `framework-core`？**  
不推荐。会缺少 Security、Redis、MyBatis-Plus 自动配置等；统一使用 `boot-starter`。

**2. `generator` 如何访问库表？**  
通过 MyBatis-Plus 与 `controller` 运行时提供的数据源，不依赖 `system` 的 Service。

**3. 依赖版本在哪里改？**  
仅改 `star-pivot-dependencies/pom.xml` 的 `<properties>` 或 `<dependencyManagement>`。

**4. 循环依赖怎么避免？**  
遵守「framework ← module ← controller」；业务模块间仅保留文档矩阵中的少量边；可用 controller 的 ArchUnit 测试约束分层。

**5. 多数据源如何用？**  
引入 `boot-starter` 即包含 `framework-datasource`；按该模块 README/配置启用 `@DS` 与 YAML 多库声明。

以上为当前仓库的依赖引用梳理，随模块增减请同步更新本文档与 BOM 登记。
