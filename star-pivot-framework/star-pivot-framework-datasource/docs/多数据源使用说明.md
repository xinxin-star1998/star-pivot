# StarPivot 多数据源工具包使用说明

模块 Maven 坐标：`com.star.pivot:star-pivot-framework-datasource`（版本由根工程 `star-pivot-dependencies` BOM 管理）。

通过 **路由数据源**（`AbstractRoutingDataSource`）+ **线程上下文** + **`@DS` 注解（AOP）** 实现运行时切换；各物理库使用 **HikariCP** 连接池。仅在配置中显式启用后才会注册主 `DataSource` Bean，避免影响未使用多数据源的应用。

---

## 1. 引入依赖

### 1.1 已使用 `star-pivot-framework-boot-starter`

Starter 已传递本模块，**无需再单独声明** `star-pivot-framework-datasource`。

### 1.2 未使用 Starter 的工程

在 `pom.xml` 中增加：

```xml
<dependency>
    <groupId>com.star.pivot</groupId>
    <artifactId>star-pivot-framework-datasource</artifactId>
</dependency>
```

需已引入 BOM（`star-pivot-dependencies`）或自行指定与主工程一致的版本。

---

## 2. 启用条件

在配置文件中设置：

```yaml
star-pivot:
  multi-datasource:
    enabled: true
```

- `enabled` 不为 `true` 时：不注册路由 `DataSource`，不改变 Spring Boot / Druid 的默认单数据源行为。
- `enabled` 为 `true` 时：根据下文 `datasources` 创建多个 Hikari 池，并注册 **`@Primary`** 的路由 `DataSource`，自动配置会尽量早于 Spring 与 Druid 的单数据源自动配置执行，避免重复创建默认库。

---

## 3. 配置项说明

前缀：`star-pivot.multi-datasource`

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `enabled` | boolean | `false` | 是否启用多数据源自动配置 |
| `primary` | String | `master` | 默认使用的数据源键；须存在于 `datasources` 中 |
| `datasources` | Map | 空 | 数据源名称 → 单库 JDBC 配置（见下表） |

单库配置（`datasources` 下每一项）：

| 属性 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| `url` | 是 | — | JDBC URL |
| `username` | 否 | — | 用户名 |
| `password` | 否 | — | 密码 |
| `driver-class-name` | 否 | — | 驱动类名；不填则由驱动推断 |
| `maximum-pool-size` | 否 | `10` | Hikari 最大连接数 |
| `minimum-idle` | 否 | — | Hikari 最小空闲连接 |
| `connection-timeout` | 否 | — | 获取连接超时（毫秒） |

---

## 4. 完整配置示例

```yaml
star-pivot:
  multi-datasource:
    enabled: true
    primary: master
    datasources:
      master:
        url: jdbc:mysql://127.0.0.1:3306/star_pivot?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
        username: root
        password: your_password
        driver-class-name: com.mysql.cj.jdbc.Driver
        maximum-pool-size: 20
        minimum-idle: 5
      slave:
        url: jdbc:mysql://127.0.0.1:3307/star_pivot_read?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
        username: readonly
        password: your_password
        driver-class-name: com.mysql.cj.jdbc.Driver
        maximum-pool-size: 10
```

启动前请确认：`datasources` 非空，且 `primary` 指向的键在 `datasources` 中存在；否则启动将抛出 `IllegalStateException`。

## 5. 使用 `@DS` 切换数据源

注解类型：`com.star.pivot.framework.datasource.annotation.DS`

```java
import com.star.pivot.framework.datasource.annotation.DS;

@DS("slave")
@Service
public class ReportService {

    public void alwaysSlave() { /* ... */ }

    @DS("master")
    public void forceMaster() { /* 方法级覆盖类上的 slave */ }
}
```

### 5.1 规则摘要

- **作用位置**：类、方法均可标注；**方法上的 `@DS` 优先于类上的 `@DS`**。
- **默认值**：未标注 `@DS` 或切面未生效时，线程上下文栈为空，路由使用 **`primary`** 对应的数据源。
- **`@DS` 的 `value()`**：须与 YAML 中 `datasources` 的 **键** 一致；若为空字符串，切面不会切换上下文（仍走默认逻辑）。
- **AOP 切点**：当前实现仅拦截 **`public` 方法**（`execution(public * *(..))`）。`private` / `protected` 方法上的 `@DS` **不会**被切面处理。
- **Spring 代理**：类上 `@DS` 通过 `AopUtils.getTargetClass` 解析，可兼容常见 JDK / CGLIB 代理场景。

切面顺序为 `Ordered.HIGHEST_PRECEDENCE`，意在先于事务切面执行，以便在开启事务前选定数据源。

## 6. 编程式切换（无 `@DS`）

包：`com.star.pivot.framework.datasource.DataSourceContextHolder`

- `push(String key)`：压入数据源键（嵌套调用时形成栈）。
- `pop()`：弹出栈顶键。
- `peek()`：查看当前键；空栈返回 `null`，路由走 `primary`。
- `clear()`：清空当前线程栈（慎用，避免误清其他逻辑压入的层级）。

务必在 `finally` 中成对 `pop`，避免线程池复用线程时串库：

```java
DataSourceContextHolder.push("slave");
try {
    // 访问从库
} finally {
    DataSourceContextHolder.pop();
}
```

## 7. 与 MyBatis / MyBatis-Plus、事务

- 业务使用的 `SqlSessionFactory` / `JdbcTemplate` / `DataSourceTransactionManager` 等应绑定到 Spring 容器中的 **`DataSource` Bean**（启用多数据源后即为路由数据源）。StarPivot 主工程在引入本模块并 `enabled=true` 后，通常无需再手写多份 `SqlSessionFactory`，只要全局仍使用**唯一**的主 `DataSource` 即可路由到不同物理库。
- **事务与数据源**：应在**已选定数据源之后**再开启事务（例如 `@Transactional` 在 Service 的 `public` 方法上，且该方法带 `@DS` 或由外层已 `push`）。若事务已开启再切换线程上下文，可能出现仍使用旧连接的情况，属使用顺序问题，而非模块缺陷。

---

## 8. 与 `spring.datasource`、Druid 的说明

- 启用本模块多数据源后，**各库连接池由本模块用 Hikari 创建**；与 `spring.datasource.druid.*` 等针对「单 Druid 池」的配置**不是**一一对应关系。若必须 Druid 监控、Wall 等，需在业务侧自行构建 `DataSource` 并扩展路由注册方式（当前版本未内置 Druid 工厂）。
- 自动配置通过 `@AutoConfigureBefore` 尽量早于 Spring 与 **Druid** 的单数据源自动配置，在仅存在一个自定义路由 `DataSource` 时，可避免再自动装配第二个默认 `DataSource` Bean（具体以 Spring Boot 与 Druid 版本行为为准）。

## 9. 常见问题

**Q：启用后启动报错「datasources 不能为空」**  
A：检查 `star-pivot.multi-datasource.datasources` 是否至少配置了一个数据源，且每项包含 `url`。

**Q：报错「primary 在 datasources 中不存在」**  
A：将 `primary` 改为已有键名，或在 `datasources` 中增加对应名称的配置。

**Q：加了 `@DS` 但没有切库**  
A：确认目标方法为 **`public`**；确认类已被 Spring 管理（如 `@Service` / `@Component`）；确认 AOP 生效（本模块已依赖 `spring-boot-starter-aop`）。

**Q：从库查不到最新数据**  
A：检查是否误用默认 `primary`、是否事务内复用连接、读写分离中间件与直连从库延迟等。
